# CVM++ Project Instructions

This document provides exhaustive instructions for building, running, and testing the CVM++ project, followed by a detailed walkthrough for creating and executing a sample calculator program.

---

## 1. Environment Setup & Build Instructions

### Prerequisites
To build and run CVM++, you need the following installed on your system:
- **Compiler:** A C++17 compatible compiler (e.g., GCC 9+, Clang 10+, or MSVC 2019+).
- **Build System:** CMake 3.16 or higher.
- **Build Tool:** `make` (for Linux/macOS) or `ninja`.

### Build Process
Follow these steps to perform a clean out-of-source build:

1.  **Navigate to the project root:**
    ```bash
    cd /home/rahul/projects/cvmplusplus
    ```

2.  **Create and enter a build directory:**
    ```bash
    mkdir -p build
    cd build
    ```

3.  **Generate build files with CMake:**
    ```bash
    cmake ..
    ```
    *Note: Use `cmake -DCMAKE_BUILD_TYPE=Release ..` for optimized performance.*

4.  **Compile the project:**
    ```bash
    make -j$(nproc)
    ```
    This produces the `cvm` executable in the `build/` directory.

## 2. Running the Project

### Executing CVM Scripts
You can run any `.cvm` script by passing its path to the `cvm` executable:
```bash
./cvm ../scripts/08_fibonacci.cvm
```

### Interactive REPL
To enter the interactive Read-Eval-Print Loop (REPL), run the executable without arguments:
```bash
./cvm
```
Inside the REPL, you can type commands line by line. Type `exit` to quit.

### Debugging & Introspection Flags
CVM++ provides several flags to inspect the execution pipeline:

- **Token Inspection:** See how the lexer breaks down your code.
  ```bash
  ./cvm --dump-tokens ../scripts/02_arithmetic.cvm
  ```
- **AST Inspection:** Visualize the Abstract Syntax Tree generated by the parser.
  ```bash
  ./cvm --dump-ast ../scripts/02_arithmetic.cvm
  ```
- **Bytecode Disassembly:** View the compiled bytecode instructions before they are executed by the VM.
  ```bash
  ./cvm --dump-bytecode ../scripts/02_arithmetic.cvm
  ```

The `scripts/error-cases/` directory contains scripts designed to trigger specific errors. Use these to verify the robustness of the error handling system:
- `error_div_zero.cvm`: Validates "Division by zero" runtime error.
- `error_undef.cvm`: Validates undefined variable errors.
- `error_type.cvm`: Validates type mismatch errors (e.g., adding a boolean to a string).

## 4. Sample Program Walkthrough: Simple Calculator

This section guides you through creating a simple interactive calculator that performs addition, subtraction, multiplication, and division based on user input.

### Step 1: Create the Source Code
Create a file named `calculator.cvm` in the `scripts/` directory with the following content:

```cvm
print "--- CVM++ Simple Calculator ---"
print "Enter first number:"
let a = 0
input a

print "Enter second number:"
let b = 0
input b

print "Addition:"
print a + b

print "Subtraction:"
print a - b

print "Multiplication:"
print a * b

print "Division:"
if b == 0 {
    print "Error: Cannot divide by zero!"
} else {
    print a / b
}
```

### Step 2: Compile to Bytecode
To see how the compiler translates this logic into VM instructions, run:
```bash
./cvm --dump-bytecode ../scripts/calculator.cvm
```

**What happens under the hood:**
1.  **`PUSH_CONST`**: The string "Enter first number:" is pushed onto the stack.
2.  **`PRINT`**: The VM pops the string and prints it.
3.  **`INPUT`**: The VM waits for user input, converts it to a `Value`, and pushes it.
4.  **`DEFINE_GLOBAL`**: The value is popped and stored in the global variable `a`.
5.  **`ADD/SUB/MUL/DIV`**: When performing arithmetic, the VM pushes `a` and `b` onto the stack, pops them to perform the math, and pushes the result back.

### Step 3: Execute on the Virtual Machine
Run the program normally:
```bash
./cvm ../scripts/calculator.cvm
```

**Interaction Example:**
```
--- CVM++ Simple Calculator ---
Enter first number:
> 10
Enter second number:
> 5
Addition:
15
Subtraction:
5
Multiplication:
50
Division:
2
```

### Step 4: Verification
Verify that the `if b == 0` check works by running the program and entering `0` for the second number. The VM should execute the `JUMP_IF_FALSE` instruction to skip the division and print the error message instead of crashing with a "Division by zero" runtime error.
